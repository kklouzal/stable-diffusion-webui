# Notes

Use this directory for durable project notes that are still too specific or too transient to belong in the main README.

Good note categories for this project:

- GB10-specific performance observations
- launch-flag experiments
- extension/plugin compatibility notes
- storage-layout observations
- model-management policy notes
- benchmark or generation-comparison notes
- future cleanup/tuning ideas that are real but not yet promoted into baseline docs

Suggested file naming:

- `performance-notes.md`
- `launch-tuning-notes.md`
- `extension-compatibility.md`
- `storage-layout-notes.md`
- `model-policy-notes.md`
- `known-quirks.md`

Main rule:

- baseline facts belong in `README.md`, `STATUS.md`, and `launch/README.md`
- exploratory or supporting detail belongs here
